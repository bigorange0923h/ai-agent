# AI Agent Bootcamp Agent Rules

本文件是本项目的 AI 协作硬规则。任何 AI 助手进入本项目后，必须先遵守这里的规则，再执行具体请求。

## 首次对话进度检查

每次新的 AI 对话开始时，必须先识别当前学习进度：

1. 检查 `days/dayXX/homework.md` 是否已经填写。
2. 检查 `days/dayXX/ai-review.md` 是否已经评审并通过。
3. 检查 `days/dayXX/reflection.md` 和 `days/dayXX/tomorrow.md` 是否已经填写。
4. 找到第一个未完成、未评审或未通过的 Day，作为当前进度。
5. 先提示：
   - 当前进度：Day X
   - 未完成事项：xxx
   - 建议下一步：xxx

如果无法访问文件，必须说明缺少哪些内容，并要求用户提供对应 Day 的 `homework.md`、`ai-review.md` 或任务文件。

可优先运行：

```bash
bash scripts/check-progress.sh
```

## AI 协作边界

- AI 不能默认直接代写当天作业、代码、反思或项目交付物。
- AI 可以解释概念、拆解任务、提出检查清单、评审作业和指出问题。
- AI 可以直接生成每天的学习资料包，包括学习目标、核心概念讲解、后端工程类比、推荐学习顺序、小练习、示例代码、检查清单、常见误区和面试追问。
- AI 可以优化用户已经写好的 `homework.md`，但必须保留用户自己的理解过程。
- 只有用户明确说明“我已经理解了，请直接帮我改 / 补 / 完成”，AI 才可以直接修改或生成完整内容。
- 如果 AI 给出参考答案，必须标记为“参考版本”，并要求用户用自己的话复述或改写。
- AI 不能把自己直接生成、用户未理解的内容作为当天通过依据。
- AI 不能直接生成并代替提交 `homework.md`、`reflection.md`、自评分或声称用户已经理解 / 已经完成的内容。

## 每日完成标准

一个 Day 通过必须同时满足：

- `homework.md` 已由用户自己填写。
- `ai-review.md` 有 AI 评审结果。
- 评审总分达到 80 分以上，并明确允许进入下一天。
- `reflection.md` 已填写当天反思。
- `tomorrow.md` 已填写明日准备。
- 当天要求的代码、图示或文档产出已经放到对应目录。

## 安全与合规

- 只能使用模拟文档、公开开源项目文档或脱敏数据。
- 禁止提交真实客户资料、数据库连接信息、内网地址、密钥、合同、生产日志等敏感内容。
- 如果发现敏感内容，必须先提示删除或脱敏，再继续处理。
